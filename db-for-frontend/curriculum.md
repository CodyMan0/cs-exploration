# 전체 커리큘럼 로드맵

프론트엔드 개발자가 백엔드 회의에서 구조적 의견을 낼 수 있게 되는 것이 목표.
모든 예시는 **병원 예약 서비스** 도메인으로 통일합니다.

> 원래 5단계로 짰던 커리큘럼에서 한 가지를 바꿨어요:
> **인덱스(검색/성능)를 동시성보다 먼저** 둡니다.
> 이유 → Lock이 "무엇을" 잠그는지가 인덱스 구조에 달려 있어서,
> 인덱스를 먼저 알아야 "왜 이 UPDATE가 옆 행까지 막지?"가 풀려요.

---

## 1단계. 구조 — 데이터의 뼈대 세우기

**학습 목표:** 데이터를 왜, 어떻게 쪼개고 묶는지 감을 잡는다.

- **PK / FK** — 데이터의 주민등록번호, 그리고 테이블 간 연결고리
- **정규화(1NF~3NF)** — 백엔드는 왜 자꾸 테이블을 잘게 쪼갤까 (중복의 위험성)
- **역정규화 & JSONB** — 쪼개는 게 항상 정답은 아니다. 유연한 문서로 퉁치는 실전 노하우

🎯 **퀘스트:** "예약 엑셀 시트 한 장"을 정규화해서 3개 테이블로 쪼개기
📁 `01-structure/`

---

## 2단계. 무결성 — 절대 깨지지 않는 약속

**학습 목표:** 에러가 나도 데이터가 반쯤 망가지지 않는 원리를 이해한다.

- **ACID** — All or Nothing. DB가 목숨처럼 지키는 4가지
- **Commit / Rollback** — "예약 + 결제"가 중간에 실패하면 둘 다 없던 일로
- **격리 수준(Isolation Level)** — Read Committed vs Repeatable Read *(원래 커리큘럼에 추가)*
- **프론트 매핑** — 트랜잭션 ↔ `try/catch`, 낙관적 업데이트(Optimistic UI)와의 관계

🎯 **퀘스트:** 일부러 실패하는 예약 트랜잭션을 ROLLBACK으로 막기
📁 `02-integrity/`

---

## 3단계. 인덱스 / 검색 — API 응답 속도의 비밀

**학습 목표:** "이 API 왜 느려요?"에 근거를 갖고 답할 수 있다.

- **인덱스 원리** — 책 뒤의 '찾아보기' 색인 (B-Tree 가볍게)
- **Full Scan vs Index Scan** — `EXPLAIN ANALYZE`로 최악과 최선을 내 눈으로
- **N+1 쿼리** — 리스트 50개에 API가 51번 쿼리하는 그 문제 *(추가)*
- **페이지네이션** — `OFFSET 10000`은 왜 느린가, 커서 기반은 왜 빠른가 *(추가)*

🎯 **퀘스트:** 느린 쿼리에 인덱스 하나 걸어서 Seq Scan → Index Scan 만들기
📁 `03-index/`

---

## 4단계. 동시성 — 수만 명이 동시에 접속한다면

**학습 목표:** 데이터가 꼬이는 걸 막는 DB의 교통정리 기술을 이해한다.

- **Lock** — "나 수정 중이니 줄 서!" (API 대기시간이 길어지는 원인)
- **Deadlock** — 서로의 Lock을 무한정 기다리다 앱이 뻗는 상황
- **MVCC** — 줄 안 서고도 안전하게 보여주는 Postgres의 필살기 (스냅샷의 마법)

🎯 **퀘스트:** 두 세션으로 같은 예약 슬롯을 동시에 잡아보고 Lock 관찰하기
📁 `04-concurrency/`

---

## 5단계. 공간 데이터 — 내 주변 2km 병원 찾기

**학습 목표:** 지도 기반 검색이 DB 내부에서 어떻게 빠르게 도는지 안다.

- **PostGIS** — Postgres를 지리 정보 DB로 만드는 확장
- **공간 인덱스(GiST)** — 위/경도에서 "반경 2km"를 어떻게 0.0x초에 찾나
- **거리 계산** — `ST_DWithin`, `geography` 타입

🎯 **퀘스트:** 강남역 좌표 기준 2km 이내 병원을 가까운 순으로 뽑기
📁 `05-postgis/`

---

## 6단계. 인프라 — 프론트와 맞닿은 실전

**학습 목표:** DB 앞단의 인프라 요소를 프론트 경험과 연결해 이해한다.

- **Connection Pool** — DB로 가는 톨게이트. 서버는 왜 연결 수를 제한할까
- **캐싱(Redis)** — DB 앞의 방파제. SWR / React Query 캐싱과 뭐가 같고 다른가

🎯 **퀘스트:** 병원 리스트 조회를 캐시로 감쌌을 때의 흐름을 그려보기
📁 `06-infra/`
